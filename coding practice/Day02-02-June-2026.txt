Module 1 : Ansible Practice 

Description: There are three parts to this playbook , 
1st is Play header : this defines the overall goal , i.e installing Nginx on Pathnex servers

2nd is the Task1 Section for APckage Installation: Here the action is named as Install Nginx , and the package is installed using YUM package manager , the software is named NGINX and STATE: present , which basically tells it to chekc if Nginx is present or not , if yes install no action if not Install it .

3rs is TASK 2 Section : Service mAnagement :  Name is Enable Naginx , its uses the Built in Service manager like systemd or init.d , the state is STARTED to make sure NGINX is up and running , and ENABLED is YES so that NGINX remain running even after software restarts 

-------------

- name: Install and start Nginx on Pathnex Servers
  host: all     #denotes all servers mentioned in the host file 
  become: yes


  task:
    - name: Install Nginx
      yum: 
        name: nginx
        state: present


    - name: Enable nginx
      service:
         name: nginx
         state: started
         enabled: yes

 and the region as US-EAST-1

Block 3 : COnfig Arguments : Here the AMI ( image type ) is defined and the Instance type is defined as r5.2xlarge

tags are the key value pair or identity assigned to track or identify the re
sources


----------------------------------------------------

  region = "us-east-1"
}

resource "aws_instance" "PathnexEC2" {
  ami           = "ami-0abcd1234abcd1234"
  instance_type = "r5.2xlarge"

  tags = {
    Name        = "Pathnex-Server"
    Environment = "Training"

MODULE 3 : KUBERNETES 


This Block denotes Kubernetes Deployment manifest YAML file , where a deployment is getting created with the folowing details 

the Object type is Deployment
Name of the deployment is pathnex-deployment
ther would be 2 PODS
the pod has 1 container named APP and image type is NGINX
the container port is 80 and the protocol is TCP
the pods and deployment are tied with the label type pathnex-app

apiVersion: apps/v1
kind: Deployment
metadata:
  name: pathnex-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: pathnex-app
  template:
    metadata:
      labels:
        app: pathnex-app
    spec:
      containers:
        - name: app
          image: nginx
          ports:
            - containerPort: 80
