MODULE 1 : Ansible 


This is an Ansible playbook that automates copying a webpage file and restarting the web server.

become: yes: Runs all tasks with administrative (root/sudo) privileges.


copy task: Overwrites the remote destination file with a local file named index.html.

notify & handlers: Triggers a restart of the Nginx service only if the index.html file actually changes.

---

- name: Manage Nginx
  hosts: all
  become: yes

  tasks:
    - name: Update config File 
       copy:
         src: index.html
         dest: /usr/share/nginx/html/index.html
       notify: Restart nginx

  handlers:
    - name: Restart Nginx
      service: 
        name: nginx
        state: restarted

MODULE 2 :  TERRAFORM :

This is a Terraform configuration block that provisions a high-performance virtual machine in AWS

aws_instance : sets the ec2 name as pathnexEC2

instance type : c6a.12xlarge ,

user_data: runs a startup shell script that install and strt apapche httpd server upon start of ec2 .


---

resource "aws_instance" "PathnexEC2" {
  ami           = "ami-0abcd1234abcd1234"
  instance_type = "c6a.12xlarge"

  user_data = <<EOF
  #!/bin/bash
  yum install -y httpd
  systemctl start httpd
  EOF
   

  tags = {
    Name = "Pathnex-EC2"
  }
}

MODULE 4 : Jenkins 


this Jenkins declarative pipeline automates a CICD process using global variables .

environment block : defines INSTITUTE_NAME with pathnex

Print stage : exacutes a shell script to print welcome message 

Build Stage : runs a maven build .


pipeline {
    agent any
    environment {
        INSTITUTE_NAME = "Pathnex"
    }
    stages {
        stage('Print') {
            steps {
                sh 'echo "Welcome to $INSTITUTE_NAME DevOps Training"'
            }
        }
        stage('Build') {
            steps {
                sh 'mvn clean package -Dinstitute.name=$INSTITUTE_NAME'
            }
        }
    }
}
