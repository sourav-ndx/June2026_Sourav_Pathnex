next was the concept of cronjob 

in Linux, a cron job is a scheduled automated task, and crontab (cron table) is the configuration file used to manage those schedules

crontab -e ---> open the file in edit mode
crontab -l --> lists the cron tab entries 
crontab -r ---> removes & dletes the entire corntab schedule file


it has 5 * or parameters 

* * * * * /path/script.sh 

1---minutes(0-59)
2---hour(0-23)
3---dayofmonth(1-31)
4---month(1-12)
5---dayofweek(0-6) ) 0 is Sunday

* (Asterisk): Matches every single value (e.g., a * in the minute field means "every minute")., 

(Comma): Separates distinct execution values (e.g., 15,30,45 means run at those specific minutes).

- (Hyphen): Specifies a range of values (e.g., 1-5 in the day-of-week field means Monday through Friday).
