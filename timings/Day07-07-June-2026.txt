3. Storage EBS  - EFS - S3

EBS : Elastic Block Storage 

It is the external block storage ( similar to a Hard drive ) and is usualy attched to a single EC2 instance . 
Its per AZ specific .
It can exist independently and can be detache from one ec2 and attched to other 
Does not Autoscale 

gp3  -  Default. General purpose SSD
io2 -- High IOPS databases — RDS, Oracle
st1 -- Throughput HDD — Kafka, big data sequential reads
sc1 - Cold HDD — cheapest, infrequent access


EFS : Elastic File Storage

Its a NFS service of AWS , Multiple EC2 instance mounts on the same EFS drive silultaneously .

Its region wide not AZ wide ,
It Autoscales and we pay for per gb stored .



S3 : Simple Storage Service - This is Object Storage Service of AWS . 

Its a global service , buckets are region specifcs but the namespace is global .
Its not attched to EC2 ( until recently ) and is accessed via a URL or API

It auto transitions objects into different classes based on age of retention 

Can be used for CRoss Region and Same region replication with different buckets 
Versioning provided for object storage 

Easily integrates with AWS services like lmbda ,

Auto Scaling / Groups :

Automatically add or remove EC2 instances based on demand. Always paired with ALB for Web workloads .

3 Things to  Configure ;

Launch Template — what to launch. 
AMI + instance type + SG + IAM role + user data. Think of it as the EC2 blueprint.

Auto Scaling Group (ASG) — where and how many.

Min / Max / Desired capacity
Which AZs to spread across
Which Launch Template to use

Scaling Policy — when to scale.

AWS WAF - WEB APPLICATION FIREWALL 

Filters HTTP / HTTPS at Layer 7 , blocks malicious web requests before they hit the app . We define the rules - WAF allows or block based on them .


How It Works
Web ACL — container for your rules. Attach to CloudFront or ALB.
Rules — conditions to match traffic. On match → Allow / Block / Count.
Rule Groups — reusable collection of rules.

Its the threat detections ervice of AWS 

Managed threat detection service. 

Continuously monitors your AWS account for malicious activity and suspicious behavior. 

Its like  an intelligent security analyst watching everything 24/7

-----------------------------------------------------------

AMAZON INSPECTOR :

Automatic Security scanning service of AWS . 

It scans EC2 , Lambdas , and Container Images for software vulnerabilities and unintended network operations .

GuardDuty = detects active threats happening now. 

Inspector = finds vulnerabilities before they're exploited

Once enabled it does automatic scanning always .

It Generates a RISK score ( not severity ) priotized by severity .

Inspector findings ≠ active attack — it's potential risk. 
GuardDuty finding = something bad happening right now.
