Port MAPPING :

docker run -d -p 8080:3000 node-app:v1

maps port 3000 of the container to 8080 of the host .

docker run -d -p 127.0.0.1:8080:3000 node-app:v1 
restrict access only from the local machine 

docker run -d -p :3000 node-app:v1
add any random host port to the container port 3000

Docker Networking :

Bridge : Creates a private internal network on the host computer where containers can securely talk to each other.

Overlay : pans a single secure network across multiple physical servers or cloud virtual machines. acts like a virtual bridge that connects containers sitting on completely different physical computers,

NONE : disables all external network interfaces. It leaves the container with only the local loopback interface (127.0.0.1 / localhost).

HOST : it removes all network isolation between the container and your computer, the container directly hijacks your host's IP address and ports. If you try to run a second container on that same port, it will crash with a "port already in use" error.
